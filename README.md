# Seeed Vending System - Full Backend + Vending-Machine Frontend

This is the complete project folder based on the uploaded `vending-machine-system-wio-full` zip.

In this version:

- `backend-full/` is the CSV-backed Node.js backend and browser admin dashboard.
- `frontend-vending-machine/wio_backend_full_reader/` is the Wio Terminal vending-machine frontend.
- `rfid-write-station/wio_rfid_writer/` is the Wio/XIAO RFID writer station sketch used to write order payloads to cards.

## Folder roles

```text
backend-full/
  Deploy this folder to Render or run it locally with Node.js.
  It stores data in CSV files under backend-full/data/.

frontend-vending-machine/wio_backend_full_reader/
  Upload this Arduino sketch to the vending-machine Wio Terminal.
  This is the customer-facing vending machine frontend.

rfid-write-station/wio_rfid_writer/
  Upload this Arduino sketch to the RFID writer Wio/XIAO station.
  It writes the order payload generated by backend-full to RFID cards.
```

## Backend deployment on Render

Create a Render Web Service with:

```text
Root Directory: backend-full
Build Command: npm install
Start Command: npm start
```

After deployment, test:

```text
https://YOUR_RENDER_URL/health
```

It should return:

```text
OK
```

The backend dashboard/admin page is:

```text
https://YOUR_RENDER_URL
```

## Vending-machine frontend setup

Open:

```text
frontend-vending-machine/wio_backend_full_reader/wio_backend_full_reader.ino
```

Change these lines:

```cpp
const char* WIFI_SSID = "YOUR_WIFI_NAME";
const char* WIFI_PASSWORD = "YOUR_WIFI_PASSWORD";
const char* BACKEND_BASE_URL = "https://YOUR_RENDER_URL";
```

Example:

```cpp
const char* WIFI_SSID = "SEEED-MKT";
const char* WIFI_PASSWORD = "your_password";
const char* BACKEND_BASE_URL = "https://your-app-name.onrender.com";
```

Do not add a slash at the end of `BACKEND_BASE_URL`.

## Runtime flow

```text
1. Operator opens backend dashboard.
2. Operator creates an order.
3. Backend stores the order in CSV and creates an RFID writer job.
4. RFID writer station writes the order JSON payload to an RFID card.
5. Customer taps that card at the vending-machine Wio.
6. Vending-machine Wio reads the JSON payload from card blocks 4,5,6,8,9,10.
7. Vending-machine Wio calls POST /api/frontend/verify-card.
8. Backend validates order and returns the dispense plan.
9. Wio dispenses using returned servo_id and quantity.
10. Wio calls POST /api/frontend/dispense-complete to mark the order collected.
```

## Important note

This build follows the uploaded backend-full API. It supports prepaid order redemption via RFID card payload.

It does not include the older direct-purchase wallet/balance flow unless backend-full is extended later with wallet APIs.
